A Blog Framework System for Lazy People | 2015-03-04

It's not that I'm lazy it's just that I'd rather be doing fun stuff over housekeeping a blog. I also realize calling what I have here a framework is a stretch. I have the [not so] unique problem of wanting a blog incorporated into my own website on my own terms. I've checked out other frameworks but I seem to desire a simplicity and ease of use (for me the author, not necessarily you the reader) that is lacking.

What's really strange is I'm not even sure I want a blog so I guess we'll see who reads this. At best I think I'll post a snippet of code on this thing and someone might find it useful but I guarantee no one is bookmarking this nonsense for weekly pearls of wisdom and insight from me. While we're at it, lets drop the word blog and use note. I'm comfortable with note but feel like a phony calling this a blog.

So, this framework amounts to nothing more than posts written in markdown which are then compiled to html and put in place using a bash script. In fact lets drop the word framework and instead use system.

Here's a better look at my note system for lazy people:

  • I write a post in oh-so-easy to use markdown format.
  • I run a bash script that grabs the title and first sentence of every post and populates a main note menu using these post snippets.
  • Each post is compiled to html using pandoc and placed in an empty webpage and linked the note menu.
  • New content is committed and pushed to GitHub where I invariably find find a spelling or grammar mistake and repeat the process.

The end result is a system where I can host my pearls of wisdom while minimizing time spent by me maintining the thing. I can also use this system to keep my cv up to date and in one spot. Every time a user accesses my cv it's guaranteed to be up to date.
